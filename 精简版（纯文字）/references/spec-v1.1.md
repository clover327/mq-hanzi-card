# Hanzi Prototype Skill Spec v1.1

## 项目定位

一个 Claude Code skill，用户输入单个汉字，自动生成融合左民安字源考据与魏老师识字方法论的儿童字源识字卡（HTML）。

## 双引擎分工

### 左民安负责「知道什么」
- 字源依据
- 本义与引申义
- 字形演变
- 构形关联字线索

### 魏老师负责「怎么教」
- 教学组织逻辑
- 哪些关系值得教
- 如何从单字推到词句语境
- 如何从解释推到朗读、互动、自动化

## 关系模块来源规范

「字族 / 偏旁 / 亲戚字」模块允许并应当参考左民安《细说汉字》中的相关解释，用于识别：
- 字的构形来源
- 字的本义与引申义链条
- 与目标字存在构形、义项或演化关联的相关字

但系统不得直接照搬学术解释，而应将其转化为儿童识字教学可用的关系：
1. 偏旁家族：提示意义类别
2. 声旁家族：提示读音线索
3. 亲戚字 / 易混字：帮助理解、迁移与区分

最终展示以「是否有助于儿童理解、区分、迁移和进入词句语境」为筛选标准，而非以学术完整性为第一标准。

## 关系入选标准

只有满足以下条件之一，才允许进主卡：
- 能帮助理解字义
- 能帮助猜读音
- 能帮助进入高频词
- 能帮助区分易错字
- 能帮助讲清楚该字为什么这样

## 展示上限

- 高频词族：2—5 个
- 偏旁家族：3—5 个例字
- 声旁家族：3—6 个例字
- 字源亲戚字：1—3 个
- 易混字：1—3 组

## 字类型分流

### 象形字
重点：本义图像、画面感、意义演变、易混字
不强制生成声旁家族。

### 会意字
重点：组成部分各自表义、合起来的意义、动作模拟

### 形声字
重点：偏旁提示义、声旁提示音、同偏旁家族、同声旁家族
关系模块最完整展开的类型。

### 假借 / 转注
儿童端保留最稳的一条主线。

## 字卡模块列表

1. Hero 头图
2. 字源故事
3. 字形演变
4. 意义旅行
5. 字族 / 偏旁 / 亲戚字
6. 字 → 词 → 句
7. 我来读（录音）
8. 亲子互动
9. 爸妈 3 分钟带法
10. 来源说明

## 生成流程

1. 识别字基本信息
2. 抽取左民安字源信息
3. 建立关系池
4. 按教学价值筛选展示关系
5. 生成儿童化字源故事
6. 生成意义旅行
7. 生成字族 / 偏旁 / 亲戚字
8. 生成字 → 词 → 句
9. 生成亲子互动 + 家长带法
10. 截图字形演变
11. 生成头图
12. 渲染 HTML

## 质量标准

- Q1：字源正确性
- Q2：儿童可理解性
- Q3：魏老师方法论完整性
- Q4：关系模块合理性
- Q5：家长可操作性
- Q6：页面可用性

## 成功标准

- 孩子知道这个字为什么这样
- 孩子不只认字，还能说词、读句
- 家长 3 分钟知道怎么带
- 这个字连到一个小家族，而不是孤立的
- 左老师提供依据，魏老师提供教学路径
- 关系不堆砌，但足够帮助理解、区分和迁移
